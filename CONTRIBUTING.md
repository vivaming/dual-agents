# Contributing

Thanks for your interest in contributing.

## Contribution Flow

- Submit changes by pull request.
- Do not merge directly to `main`.
- Wait for maintainer review and approval before merging.

## Expectations

- Keep changes scoped and easy to review.
- Add or update tests for behavior changes.
- Avoid committing secrets, machine-specific paths, or private project context.
- Preserve `LICENSE` and `NOTICE` in redistributions and substantial forks.

## Pull Requests

Please include:

- what changed
- why it changed
- how it was tested
- any risks or follow-up items

## Attribution

If you reuse substantial portions of this project in a redistribution or public fork, preserve the Apache 2.0 license and the `NOTICE` file so the original source remains attributed.
